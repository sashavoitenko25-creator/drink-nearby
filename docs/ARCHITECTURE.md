# Drink Nearby — Architecture

Version: 1.0

---

# О проекте

Drink Nearby — это карта живых знакомств.

Пользователь появляется на карте только тогда, когда действительно хочет встретиться.

Нет лайков.

Нет ожидания.

Нет переписок.

Открыл карту → увидел человека → построил маршрут → встретился.

---

# Главная идея

Все действия должны происходить максимально быстро.

Открытие приложения:

до 3 секунд

Появление людей:

до 5 секунд

Построение маршрута:

1 нажатие

---

# Главные принципы

## Один файл = одна ответственность

Нельзя делать файл, который одновременно:

- рисует интерфейс
- делает запросы
- работает с картой
- управляет состоянием

Каждый файл отвечает только за одну задачу.

---

## Минимум зависимостей

Любой модуль должен знать только то, что ему необходимо.

Например:

RouteController

не должен знать ничего про ActivePanel.

---

## Минимум дублирования

Любая логика должна существовать только в одном месте.

Если одинаковый код появился дважды — его нужно вынести.

---

## Красивый код важнее количества кода

Лучше написать:

50 строк

которые легко понять,

чем

15 строк,

которые невозможно читать.
